# 方法
1. Red 動かない
   失敗するコードを書く
2. Green 動く・汚い
   テストが通るコードを書く
3. Refactor 動く・きれい
   整理する

# テスト駆動

1つに集中して各個撃破

* Red
    それなりに設計する
* Green にする
    動作するが正義
* Refactor にする
    テストを通しながらひたすらきれいにする
